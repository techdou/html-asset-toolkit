# React / Vue Static Build Packaging

Use this reference when packaging React, Vue, Vite, Vue CLI, Create React App, webpack, or similar static frontend build outputs into a single offline HTML file.

## Recommended Agent path

Prefer the wrapper script when the user asks to build and package a frontend project:

```bash
cd my-app
python /path/to/html-asset-toolkit/scripts/package_frontend_build.py .
```

Use `--skip-build` when the build directory already exists:

```bash
python /path/to/html-asset-toolkit/scripts/package_frontend_build.py . --skip-build
```

The wrapper looks for production build entries in this order:

```text
dist/index.html
build/index.html
out/index.html
```

`public/index.html` is not auto-selected because it is commonly a source template in React/Vue/Vite projects. Package it only when explicitly requested with `--entry public/index.html`.

## Manual workflow

Vite / Vue CLI / common dist output:

```bash
npm run build
python /path/to/html-asset-toolkit/scripts/inline_assets.py dist/index.html --preset react-vue-build
python /path/to/html-asset-toolkit/scripts/validate_single_html.py dist/index.single.html
```

Create React App:

```bash
npm run build
python /path/to/html-asset-toolkit/scripts/inline_assets.py build/index.html --preset create-react-app
python /path/to/html-asset-toolkit/scripts/validate_single_html.py build/index.single.html
```

Custom build entry:

```bash
python /path/to/html-asset-toolkit/scripts/package_frontend_build.py . --entry path/to/index.html --skip-build
```

## What gets embedded

The inliner handles build references in HTML:

```html
<script type="module" crossorigin src="/assets/index-xxxxx.js"></script>
<link rel="stylesheet" crossorigin href="/assets/index-xxxxx.css">
<link rel="icon" href="/favicon.ico">
```

It recursively processes CSS and JavaScript content:

```css
@import "./theme.css";
.hero { background: url('/assets/bg-xxxxx.png'); }
@font-face { src: url('/assets/font-xxxxx.woff2'); }
```

```js
const img = "/assets/logo-xxxxx.svg";
const model = "/assets/model-xxxxx.glb";
import("/assets/chunk-xxxxx.js");
```

It also detects static template literals generated by Vite/esbuild minification:

```js
var Kg=[`/buildings/buildingA.jpg`,`/buildings/buildingB.jpg`];
o&&(o.src=Kg[e]||Kg[0]);
```

Dynamic template literals remain intentionally unresolved because the filename is computed at runtime:

```js
const img = `/buildings/${buildingName}.jpg`;
```

For dynamic cases, convert the app to a static lookup table or bundler imports before packaging.

## Output locations

Build entries are written beside the entry file:

```text
my-app/dist/index.html  -> my-app/dist/index.single.html
my-app/build/index.html -> my-app/build/index.single.html
my-app/out/index.html   -> my-app/out/index.single.html
```

This avoids accidental nested outputs such as `dist/dist/index.single.html`.

## Root-relative URLs

Frontend build tools often emit browser-root-relative URLs:

```text
/assets/app.js
/static/js/main.js
/favicon.ico
```

For packaging, these are resolved under the static site root. By default, the root is the input HTML directory.

Example:

```bash
python scripts/inline_assets.py dist/index.html
```

Resolves:

```text
/assets/app.js -> dist/assets/app.js
/favicon.ico   -> dist/favicon.ico
```

Use `--root-dir` only when the HTML entry and static root are not the same directory:

```bash
python scripts/inline_assets.py custom/path/index.html --root-dir dist
```

## SRI and crossorigin

`integrity` attributes are removed by default because the content changes after CSS/JS references become Data URLs. Leaving stale SRI hashes can break script/style loading.

`crossorigin` attributes are usually harmless but unnecessary for embedded Data URLs. The tool currently preserves them unless validation indicates a browser issue.

## SPA routing caution

Single-file packaging is best for demos and offline teaching. It does not replace a real server for production SPA routing. If the app depends on history-mode routes, API calls, authentication redirects, service workers, or CDN-hosted runtime files, the single HTML may still need design changes.

For demos, prefer hash routing or a single landing route.

## Known limits

- External CDN resources are not fetched or embedded by default.
- API responses are not embedded.
- Service workers and PWA caches are not required for offline demo packaging and may interfere with local testing.
- Very large MP4/GLB/STL/PDF resources can produce slow-loading HTML.
- Source maps should normally be excluded from the final teaching artifact.

## Agent checklist

1. Confirm whether the user wants to build from source or package an existing build folder.
2. Run `package_frontend_build.py .` for a full build-and-package flow.
3. Use `--skip-build` if the build output already exists.
4. Validate the generated HTML. The validator checks both surface HTML and decoded embedded CSS/JS Data URLs for remaining local references.
5. Report `index.single.html`, manifest path, and warnings.
